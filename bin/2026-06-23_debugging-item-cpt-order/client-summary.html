Auction Item Image Ordering

A plain-English summary of what we found, what it means, and the plan.  ·  June 2026

What you noticed

On the auction results pages, a small number of lots were showing the wrong lead photo as their thumbnail — and within those lots, the photos could appear out of order. For example, Lot 274 was showing its 5th photo instead of the intended first one.

Why it happens

Every lot's photos are arranged by a stored "order number" (1st, 2nd, 3rd…). For most images that number is set correctly when the auction is imported. But for a small fraction of photos, the import process left the order number blank. When several photos in a lot have a blank order, the website can't reliably tell which one should come first — so it sometimes picks the wrong photo for the thumbnail.

This is a long-standing quirk, not something that recently broke. We checked the last several auctions, and every one has a small number of affected photos — going back to 2023. It has always been a tiny percentage; it just became visible on a specific lot.

How big is it?

In the current 2026 Summer auction:

Photos in the auction~18,500 across 1,320 lots
Photos with a blank order number~34 (about 0.2%)
Lots affected~14
Everything elseCorrect

Recent auctions look similar — typically a few dozen photos each. It's a small, steady trickle, never a flood.

The good news

  • Already works  Reordering a lot's photos by dragging them in the Add Media window permanently fixes that lot. Staff can correct any lot by hand today.
  • Built & tested  We created a safe tool that automatically restores the correct order for the clearly-fixable cases — 13 lots / 33 photos in this auction — and flags the small handful that need a quick human decision.
  • Careful  Nothing has been changed yet. Everything so far has been measured without touching your data, and the fix is designed to never overwrite ordering your staff set by hand.

The plan

  1. Apply the automatic fixes for the clearly-fixable lots (with your go-ahead) — starting with one lot so you can see the result, then the rest.
  2. Hand-fix the few flagged lots — a handful that need a person to choose the right first photo (about a minute each).
  3. Add an in-admin report so your team can see affected lots for any auction and fix them with one click, any time.
  4. Strengthen the import process so it stops leaving photos without an order number in the first place — addressing the root cause, not just the symptoms.

Bottom line

A small, long-standing import quirk left a fraction of photos without an order number, which occasionally surfaced as the wrong thumbnail. It's well understood, the bulk is auto-fixable, the rest is a quick manual touch, and we're adding tools plus an import fix so it stops recurring.
