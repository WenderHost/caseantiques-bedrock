How this works (the per-folder lifecycle)
  1. pending → not yet copied.
  2. copied (Phase 1) → Claude runs aws s3 sync wp-content → app/uploads and verifies (object count + size match, --dryrun = 0). Non-destructive; visitors still served from wp-content via the nginx rule.
  3. cutover (Phase 2) → folder removed from the legacy alternation in cdn-redirect-legacy-paths.conf + nginx reload; now served from app/uploads. You deploy nginx.
  4. deleted (Phase 3) → wp-content copy removed. Destructive — only with explicit sign-off, done last after the dust settles.
